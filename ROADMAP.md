# ROADMAP

## Phase 0 / v0.x

Current focus:

- append-only transaction ledger,
- transaction-bound evaluations,
- deterministic score derivation,
- anti-collusion hardening,
- A2A-extension-compatible score exposure.

This is the current ledger-first trust substrate prototype.

## Hardening pass

Near-term hardening work includes:

- README / docs / tests alignment,
- explicit semantic boundary documents,
- stronger demo coverage,
- clearer separation between substrate semantics and derived views.

## v0.2

Planned next-step scoring work:

- evaluator-weighted scoring,
- trust-propagation-aware weighting,
- more robust handling of sparse evidence.

This is where PageRank-style or evaluator-weighted approaches belong if added.

## Future directions

Future work may include:

- cryptographic attestations,
- portable verification artifacts,
- provider-neutral trust attestation mapping,
- richer A2A trust extension alignment,
- graph-based or scoped trust composition.

These are explicitly future layers, not present guarantees of the current prototype.
